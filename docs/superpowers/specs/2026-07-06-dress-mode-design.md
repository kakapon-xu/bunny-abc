# Bunny ABC 穿衣服模式设计文档

> 版本: v1.0
> 日期: 2026-07-06
> 状态: 设计中

## 一、产品定位

在现有 FIND IT（听音找字母）模式基础上，新增"穿衣服"主题场景，通过"给小人穿衣服"的可视化进度，增强 2-3 岁宝宝的成就感和学习动力。

每轮游戏围绕一个天气/场景主题（泳池、雪天、雨天、晴天）展开，宝宝每答对一道字母题，小人就穿上一件对应主题的衣服，全部穿完后触发主题专属庆祝动画。

### 设计原则
- **大图片、小文字**：除了目标字母醒目显示外，其他文字尽量小，以视觉画面为主
- **常识正确**：每个主题的穿搭顺序符合真实生活逻辑，不误导小朋友
- **零挫败感**：答错不脱衣服，永远鼓励，可以无限次尝试
- **可扩展**：主题系统设计为可插拔，后续可轻松添加"吃饭""刷牙"等新主题

## 二、游戏玩法

### 整体流程
```
选择"穿衣服乐园"场景 → 选择 FIND IT 模式
    ↓
随机选主题（泳池/雪天/雨天/晴天）→ 显示主题介绍卡片（2秒）
    ↓
第1题：听发音找字母 → 答对 → 穿上第1件衣服 → 播放单词发音
    ↓
第2题：听发音找字母 → 答对 → 穿上第2件衣服
    ↓
... 共8题 ...
    ↓
全部穿完 → 主题庆祝动画（3秒）
    ↓
自动下一轮 → 新的随机主题
```

### 答对反馈
- 小人穿上新衣服（"嗖"的出现动画）
- 小人开心蹦一下
- 小兔子开心表情 + 表扬语音（"Great job!" / "Wow!"）
- 播放对应单词发音
- 0.8 秒后自动进入下一题

### 答错反馈
- 小兔子温柔鼓励（"Try again!"）
- **不脱衣服**，保持当前进度
- 1.2 秒后恢复等待状态，可继续尝试

### 重听功能
- 点击小兔子头像 / 点击重听按钮 / 按空格键 → 重复播放题目

## 四、界面设计

### 4.1 整体布局

```
┌─────────────────────────────────────────────────────┐
│  ← 返回    👗 穿衣服乐园         🔊              │  顶部栏（文字小）
├──────────────────┬──────────────────────────────────┤
│                  │                                  │
│   🧍小人区域     │    🐰 小兔子 + 对话气泡          │
│   (大号，占1/3)  │    "Can you find S?"             │
│                  │    字母S超大醒目显示             │
│   进度: 3/8      │    [🔊重听]  [👀提示]            │
│   (小文字)       │    (按钮图标为主，文字小)        │
│                  │                                  │
├──────────────────┴──────────────────────────────────┤
│                                                     │
│              虚拟键盘 (大按键)                       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4.2 设计要点
- **小人区域**：左侧 1/3 宽度，是页面的视觉焦点
- **目标字母**：在对话气泡中超大显示（120px+），粉色粗体，弹跳动画
- **其他文字**：尽量小（14-16px），颜色浅，不抢画面
- **按钮**：以图标为主，文字为辅
- **键盘**：保持现有大小，大按键适合小手指

### 4.3 主题介绍卡片
每轮开始时全屏显示主题卡片：
- 大 emoji 图标（如 ❄️）
- 主题名称中英文（"雪天来啦！/ Snow Day!"）
- 2 秒后自动消失，开始游戏

### 5.3 数据结构

```typescript
// 衣服位置类型
type ClothingPosition = 
  | 'head'        // 头部（帽子、泳帽等）
  | 'eyes'        // 眼睛（眼镜、泳镜等）
  | 'neck'        // 脖子（围巾）
  | 'body-inner'  // 身体内层（秋衣）
  | 'body-mid'    // 身体中层（毛衣）
  | 'body-outer'  // 身体外层（外套、雨衣）
  | 'legs-inner'  // 下身内层（秋裤）
  | 'legs-outer'  // 下身外层（裤子、短裤）
  | 'hands'       // 手（手套）
  | 'feet'        // 脚（鞋子、雨鞋）
  | 'accessory';  // 配件（雨伞、书包、救生圈等）

// 单件衣服
interface ClothingItem {
  id: string;
  letter: string;           // 对应字母
  word: string;             // 英文单词
  wordCn: string;           // 中文释义
  emoji: string;            // emoji 图标
  position: ClothingPosition;
  zIndex: number;           // 渲染层级（内层小，外层大）
}

// 庆祝动画类型
type CelebrationType = 'swim' | 'ski' | 'puddle' | 'sand';

// 主题
interface DressTheme {
  id: string;
  name: string;             // 中文名称
  nameEn: string;           // 英文名称
  emoji: string;            // 主题 emoji
  startOutfit: string;      // 起始状态描述（用于渲染初始小人）
  startEmoji: string;       // 起始状态 emoji
  bgGradient: string;       // 背景渐变色（CSS gradient）
  items: ClothingItem[];    // 8件衣服，按穿戴顺序排列
  celebration: CelebrationType; // 庆祝动画类型
}
```

### 5.4 核心逻辑

#### 状态管理
```typescript
const [currentTheme, setCurrentTheme] = useState<DressTheme>(null);
const [currentItemIndex, setCurrentItemIndex] = useState(0); // 当前要穿第几件
const [wornItems, setWornItems] = useState<ClothingItem[]>([]); // 已穿上的衣服
const [gamePhase, setGamePhase] = useState<'intro' | 'playing' | 'celebrating'>('intro');
const [targetLetter, setTargetLetter] = useState('');
```

#### 流程控制
1. **进入页面** → 随机选主题 → 进入 intro 阶段 → 2秒后开始第1题
2. **答对** → wornItems 添加当前衣服 → 播放单词发音 → 0.8秒后 currentItemIndex + 1 → 下一题
3. **全部穿完**（currentItemIndex === 8）→ 进入 celebrating 阶段 → 播放庆祝动画 → 3秒后重新选主题，开始新一轮
4. **答错** → 播放鼓励语音 → 不改变状态，继续等待
