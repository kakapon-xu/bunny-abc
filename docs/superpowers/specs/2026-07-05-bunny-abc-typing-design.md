# Bunny ABC - 儿童英语启蒙打字软件 设计文档

> 版本：v1.0  
> 日期：2026-07-05  
> 目标用户：2-3岁儿童  
> 核心目标：通过打字互动的方式，进行英语字母和简单单词的启蒙学习

---

## 1. 产品概述

### 1.1 产品定位

一款面向2-3岁儿童的英语启蒙软件，以可爱的小兔子为伙伴，通过"按键盘"这一互动方式，让孩子在游戏中认识26个英文字母和常见英语单词。

打字是互动载体而非学习目标——孩子通过按键触发有趣的视觉和声音反馈，自然地接触英语。

### 1.2 设计原则

- **低压力无惩罚**：没有失败、没有扣分、没有Game Over，只有鼓励
- **即时反馈**：每一次按键都有视觉+听觉双重反馈
- **大而简单**：大按钮、大点击区域、简单清晰的交互逻辑
- **短时长**：单次5-10分钟，10分钟后自动休息提醒
- **全平台**：响应式设计，电脑/平板/手机都能玩

### 1.3 技术选型

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | React + TypeScript | 组件化开发，类型安全 |
| 样式 | 纯CSS + CSS动画 | 轻量流畅，儿童动画效果 |
| 语音 | Web Speech API | 浏览器原生语音合成，无需音频文件 |
| 图片 | SVG + Emoji | 矢量清晰，加载快 |
| 适配 | 响应式CSS | 一套代码全设备 |
| 部署 | GitHub Pages | 免费静态托管 |

## 2. 整体架构

### 2.1 页面导航流程

```
主页(Home) → 选场景(SceneSelect) → 选模式(ModeSelect) → 游戏(Game)
     ↑              ↑                      ↑                  │
     └──────────────┴──────────────────────┴──────────────────┘
                          随时可返回
```

### 2.2 页面说明

| 页面 | 功能 |
|------|------|
| **主页 Home** | 小兔子迎接，一个大"Play"按钮开始游戏 |
| **选场景 SceneSelect** | 6个主题场景卡片，全部开放，任选 |
| **选模式 ModeSelect** | 进入场景后，选择"随便看看"或"找一找"两种模式 |
| **游戏页 Game** | 核心玩法页面，两种模式各自的游戏界面 |

### 2.3 选模式页 (ModeSelect)

进入某个场景后，显示两个大按钮供选择：

- 🔍 **随便看看 (Explore)** — 自由探索，按什么字母都有惊喜
- 🎯 **找一找 (Listen)** — 听发音，找到正确的字母

两个按钮并排显示，各占一半宽度，按钮上有图标+英文名称+简短说明。顶部有返回按钮可以回到场景选择页。

## 4. 游戏模式

每个场景内有两种游戏模式。

### 4.1 模式一：随便看看 (Explore Mode)

**定位：** 自由探索、低压力、大量输入

**玩法流程：**
1. 进入场景，小兔子站在画面中
2. 屏幕底部显示字母键盘（实体键盘或虚拟键盘）
3. 孩子按下任意字母 → 触发以下反馈：
   - 🖼️ 画面中弹出对应单词的大图片
   - 🔊 播放英语单词发音
   - 🐰 小兔子做出惊喜反应（睁大眼睛、蹦跳、指向图片）
   - ✨ 闪光粒子特效
4. 可以连续按不同字母，图片和发音会切换

**设计要点：**
- 按任何字母都有正反馈，没有"错"的概念
- 图片从当前场景的单词库中匹配（按首字母）
- 如果一个字母在当前场景对应多个单词（如B同时有Banana和Bread），则随机选一个展示
- 如果字母在当前场景没有对应单词，从全局单词库随机选一个
- 图片停留2-3秒后淡出，或按新字母时立即切换

### 4.2 模式二：找一找 (Listen Mode)

**定位：** 听力练习、有目标、检验学习效果

**玩法流程：**
1. 小兔子出题：
   - 语音："Can you find A?"
   - 文字气泡显示字母
   - 旁边显示一个小图片提示（如苹果）
2. 孩子在键盘上找字母并按下
3. **答对了：**
   - 小兔子开心跳跃 + 欢呼语音 "Great job!"
   - 闪光特效
   - 2秒后自动出下一题
4. **答错了：**
   - 小兔子温柔摇头 + 鼓励语音 "Try again!"
   - 不扣分、不减少机会，可以继续按
   - 可以多次尝试直到答对

**设计要点：**
- 题目从当前场景的单词中随机抽取
- 答对后自动下一题，没有关卡概念
- 没有失败机制，可以一直玩
- 孩子可以点击图片或发音按钮重听

## 5. 小兔子角色设计

### 5.1 角色定位

小兔子是孩子的学习伙伴，兼具"任务小向导"和"被照顾的小伙伴"双重角色。

### 5.2 表情状态

| 状态 | 触发场景 | 动画 |
|------|----------|------|
| 🐰 常态 | 等待孩子输入时 | 轻轻呼吸、偶尔眨眼睛 |
| 🤩 惊喜 | 随便看看模式按到字母时 | 睁大眼睛、蹦跳、指向图片 |
| 😄 开心 | 找一找模式答对时 | 拍手跳跃、耳朵晃动 |
| 🤔 期待 | 出题后等待答案时 | 歪头、微微晃动 |
| 😌 鼓励 | 答错时 | 温柔摇头、微笑 |
| 😴 休息 | 玩满10分钟后 | 打哈欠、眼睛眯起来 |

### 5.3 英文语音台词

**随便看看模式：**
- "Wow! Apple!"
- "Look! A banana!"
- "So pretty!"
- "How cute!"

**出题时：**
- "Can you find A?"
- "Where is B?"
- "Listen... What letter is this?"

**答对时：**
- "Great job!"
- "You did it!"
- "Yes! That's right!"
- "Wonderful!"

**答错时：**
- "Try again!"
- "Almost there!"
- "Keep trying!"
- "That's okay, try again!"

**休息提醒：**
- "I'm sleepy..."
- "Let's rest our eyes!"
- "Yawn... so tired!"

## 6. 界面设计

### 6.1 视觉风格

- **色调：** 马卡龙色系，粉色为主调，明亮柔和不刺眼
- **风格：** 卡通可爱，大圆角，大按钮
- **字体：** 圆润的无衬线字体，字母清晰易读
- **布局：** 上中下结构，简洁不拥挤

### 6.2 主页 (Home)

```
┌─────────────────────────────┐
│         ☁️    ☁️            │
│                             │
│        🐰 Bunny ABC         │
│                             │
│           🐰                │
│        (小兔子)             │
│                             │
│        ▶  Play              │
│     (大按钮，居中)          │
└─────────────────────────────┘
```

- 大标题 "Bunny ABC"
- 小兔子在中央
- 一个大"Play"按钮
- 背景：柔和渐变天空+草地

### 6.3 选场景页 (SceneSelect)

```
┌─────────────────────────────┐
│  ←         Choose a scene   │
│                             │
│   🍎        🐶        🎨   │
│ 水果园    动物园    彩虹屋  │

│   🧸        🍽️        🌙   │
```

- 6个场景卡片，2行3列
- 每个卡片有图标和名称
- 全部场景可点击（无解锁机制）

### 6.4 游戏页 (Game)

```
┌─────────────────────────────┐
│  ←    🔊                    │  ← 顶部：返回+声音按钮
│                             │
│          A a                │  ← 字母气泡
│        🍎  (大图)           │  ← 单词图片
│                             │
│           🐰                │  ← 小兔子
│        (小兔子)             │
│                             │
│   Q W E R T Y U I O P       │
│    A S D F G H J K L        │  ← 虚拟键盘
│     Z X C V B N M           │
└─────────────────────────────┘
```

- **顶部栏**：返回按钮、声音开关
- **上部区域**：字母气泡 + 单词图片
- **中部区域**：小兔子角色
- **底部区域**：虚拟键盘（触屏设备显示，实体键盘设备可隐藏/显示）

### 6.5 儿童友好设计

- 所有可点击元素 ≥ 44x44px（小手指友好）
- 每一个操作都有声音反馈
- 没有广告、没有内购、没有外部链接
- 10分钟休息提醒机制
- 音量默认适中，不会突然大声吓到孩子

## 7. 技术架构

### 7.1 技术栈

- **前端框架：** React 18 + TypeScript
- **路由：** React Router (HashRouter，适配GitHub Pages)
- **样式：** 纯CSS（CSS Modules 或 全局CSS）
- **语音：** Web Speech API (SpeechSynthesis)
- **构建工具：** Vite
- **部署：** GitHub Pages

### 7.2 项目结构

```
src/
├── components/           # 可复用组件
│   ├── Bunny.tsx         # 小兔子角色
│   ├── Keyboard.tsx      # 虚拟键盘
│   ├── WordBubble.tsx    # 字母/单词气泡
│   ├── WordImage.tsx     # 单词图片
│   └── SceneCard.tsx     # 场景选择卡片
├── pages/                # 页面组件
│   ├── Home.tsx          # 主页
│   ├── SceneSelect.tsx   # 选场景
│   ├── ExploreMode.tsx   # 随便看看模式
│   └── ListenMode.tsx    # 找一找模式
├── data/                 # 数据配置
│   ├── scenes.ts         # 6个场景配置
│   ├── words.ts          # 单词库
│   └── bunnyLines.ts     # 兔子语音台词
├── utils/                # 工具函数
│   ├── speech.ts         # 语音合成工具
│   └── useTimer.ts       # 休息计时hook
├── App.tsx               # 根组件+路由
└── index.css             # 全局样式+动画
```

### 7.3 核心模块说明

#### 语音模块 (speech.ts)
- 封装 Web Speech API
- 支持美式英语发音
- 可调节语速（适合儿童的稍慢语速）
- 播放字母发音和单词发音

#### 键盘模块 (Keyboard.tsx)
- 检测设备类型，自动显示/隐藏虚拟键盘
- 实体键盘事件监听
- 触屏设备显示大按键虚拟键盘
- 按键高亮反馈

#### 计时模块 (useTimer.ts)
- 累计游戏时间
- 10分钟后触发休息提醒
- 小兔子切换到睡眠状态

### 7.4 数据结构

```typescript
// 单词
interface Word {
  id: string;
  text: string;       // 单词文本，如 "Apple"
  letter: string;     // 首字母，如 "A"
  emoji: string;      // 对应的emoji图标
  sceneId: string;    // 所属场景
}

// 场景
interface Scene {
  id: string;
  name: string;       // 场景名称
  emoji: string;      // 场景图标
  bgGradient: string; // 背景渐变色
  wordIds: string[];  // 包含的单词ID
}

// 兔子台词
interface BunnyLines {
  explore: string[];  // 随便看看时的感叹
  question: string[]; // 出题
  correct: string[];  // 答对
  wrong: string[];    // 答错鼓励
  rest: string[];     // 休息提醒
}
```

## 8. 部署方案

### 8.1 GitHub Pages 部署

- **仓库：** 公开GitHub仓库
- **构建：** `npm run build` 生成静态文件到 `dist/` 目录
- **部署方式：** GitHub Actions 自动部署，或手动推送到 `gh-pages` 分支
- **访问地址：** `https://<username>.github.io/<repo-name>/`
- **路由：** 使用 HashRouter 避免刷新404问题

### 8.2 部署优势

- 完全免费
- 自动HTTPS
- 全球CDN加速
- 纯静态，无服务器维护成本
- 版本管理方便
